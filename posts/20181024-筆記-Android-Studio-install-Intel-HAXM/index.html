在 AndroidStudio 中，使用 SDK Manager 安裝 Intel x86 Emulator Accelerator (HAXM installer) 失敗

安裝過程中，出現以下錯誤訊息

說明兩種可能原因：

  1. 系統未支援虛擬化技術(Virtualization Technology)：在 Intel CPUs 中，為 VT-x，AMD 為 AMD-V
  2. 被 Hyper-V 佔據使用了

CASE1

要確認系統是否已經開啟或支援虛擬化技術，有以下方法：

1. 工作管理員

Win8 以上

打開工作管理員＞效能頁面，右下角可看到虛擬化(Virtualization)否為啟用狀態
(如下圖)

2. 工具

Intel, AMD 有提供相關工具可以檢測與確認：

  • Intel® Processor Identification Utility - Windows* Version
  • AMD Virtualization™ Technology and Microsoft® Hyper-V™ System Compatibility Check Utility

Windows 也有提供 Microsoft® Hardware-Assisted Virtualization Detection Tool

若需要啟用，則要進入 BIOS ，找到虛擬化技術、Intel VT-x 等等類似的字樣。

CASE2

若被 Hyper-V 使用，需至控制台＞程式與功能＞開啟或關閉 Windows 功能
將 Hyper-V 關閉後，重新啟動，才能使用
